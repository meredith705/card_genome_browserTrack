Description

This track displays structural variants (SVs) from long-read ONT sequencing data produced at the NIH Center for Alzheimer's and Related Dementias (NIH CARD) Long-Read Innitiative. SVs are genomic rearrangements larger than ~50 bp, including deletions, insertions, duplications, inversions, and complex substitutions.

These SVs are called against GRCh38 from Oxford Nanopore long-read DNA sequencing of prefrontal cortex brain tissue samples produces at the NIH CARD Long-Read Innitiative. This track includes 206 samples of European Ancestry from a North American Brain Expression Consortium study (NABEC, dbGaP:phs001300) and 146 samples of African American Ancestry from the DIRP NIMH Human Brain Collection Core (HBCC:phs000979).

Display Conventions and Configuration

Items are colored by SV type:

  • DEL — deletion
  • DUP — duplication
  • INV — inversion
  • INS — insertion (shown as a 1 bp feature at the insertion point)
  • BND — breakend / translocation

The item name is SVTYPE-SVLEN (e.g. DEL-1200). Clicking an item shows its details, including:

  • svType / svLen — structural variant type and length, from the source VCF INFO fields
  • alleleFreq — allele frequency, from the source VCF INFO/AF field
  • carrierCount — number of samples with a non-reference, non-missing genotype for this variant
  • nabecCount / hbccCount — carrier counts restricted to samples whose ID starts with NABEC or HBCC, respectively

Methods

NABEC samples were sequenced with ONT R9.4.1 and HBCC samples were sequnced with R10.4.1 PromethION flow cells with median N50 of 27 Kbp and at an average depth of ∼40x genome coverage. SVs were called from minimap2 read alignments with Sniffles2 v.2.3 as well as with assembly alignments using Hapdiff. Assemblies were produced by Shasta v0.11.1 and phased using HapDup v.0.12. Assembly SVs were then merged across samples with Truvari while read SVs were merged aross samples with Sniffles2, finally Truvari was again used to merge the read and assembly SVs together as well as across the 2 cohorts. All of these data were produced on AnVIL Terra using Nanopore Analysis Pipeline (NAPU) workflows which can be found here, also see this publication for more information about the NAPU workflow. SVs were then converted to bigBed for display with a custom pipeline (source scripts).

Data Access

The bigBed file underlying this track can be downloaded directly from the hub's GitHub repository, or queried by region with the UCSC REST API or command-line bigBedToBed utility. The calls in VCF format is available on AnVIL for controlled access through dgGaP: NABEC:phs001300.v6.p1 (sub study phsID: phs003181.v2.p1) and HBCC:phs000979.v4.p2.
